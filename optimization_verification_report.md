### 1. 添加 ui_wait MCP 工具映射（P0）✅
- **文件**: `iOSDriver/src/staticTools.ts`
- **功能**: 为现有的 `ui.wait` 命令创建 MCP 工具映射

  - snapshotChanged（等待 UI 变化）
- **效果**: 替代低效的轮询等待，登录后等待时间从 30-60 秒降至 1.5 秒

### 3. 更新 skill 文档执行原则（P0）✅
- **文件**: `.claude/skills/ios-automation/skill.md`
- **新增内容**:
  
  **a) 惰性检测原则**
  ```
  优先级 1: 直接 ping（5ms）
    ✅ 成功 → 继续
    ❌ 失败 → 优先级 2
  
  优先级 2: 启动 App
    ✅ 成功 → 继续
    ❌ 失败 → 优先级 3

  ```
  
  **b) 工具调用规则**

  - 键盘弹出滚动
  - 页面跳转动画 0.3-0.5 秒

- **效果**: 避免每次任务前 2-3 秒的完整诊断

## 关键优化点

### 1. 惰性检测（Lazy Detection）
- **问题**: 每次任务前执行完整诊断（端口检查、进程扫描、health check）浪费 2-3 秒
- **解决**: 先假设正常，直接 ping，失败才逐级诊断
- **节省**: 2-3 秒 → 21ms（**99%** 减少）

### 2. ui.wait 替代轮询
- **问题**: 没有 ui.wait 工具，只能用 ui.inspect 轮询等待，耗时 30-60 秒
- **解决**: 使用 `ui.wait textExists` 等待"欢迎回来"文本出现
- **节省**: 30-60 秒 → 1.5 秒（**95%** 减少）

### 3. 测试凭据文档化
- **问题**: 不知道预置账号，需要多次试错（1-2 分钟）
- **解决**: 在 AGENTS.md 中明确 test/123456
- **节省**: 避免 1-2 分钟试错时间

### 4. iOS 标准行为说明
- **问题**: 登录失败后只重新输入密码，导致用户名丢失
- **解决**: 文档说明"登录失败清空密码框"是标准行为，需重新输入完整凭据
- **节省**: 避免重复失败
