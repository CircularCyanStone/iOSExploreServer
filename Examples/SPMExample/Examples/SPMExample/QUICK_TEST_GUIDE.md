## 📋 核心测试场景

### 场景 1：成功登录
```
测试登录功能，使用账号 test/123456，验证能否进入首页并显示正确的用户信息。
```

### 场景 2：注册新用户
```
测试注册功能：
1. 点击"去注册"
2. 填写信息：用户名 newuser，邮箱 newuser@example.com，密码 password123
3. 验证注册成功并返回登录页
4. 使用新账号登录
```

### 场景 3：重置密码
```
测试密码重置：
1. 点击"忘记密码"
2. 填写：用户名 test，邮箱 test@example.com，新密码 newpass123
3. 验证重置成功
4. 使用新密码登录
```

### 场景 4：退出登录
```
测试退出登录：点击"退出登录"按钮，确认退出，验证返回登录页。
```

## ❌ 错误处理测试

### 空输入验证
```
在登录页面不填任何内容直接点击登录，验证是否有"请输入用户名"的提示。
```

### 密码错误
```
输入用户名 test 和错误密码 wrongpass，验证是否显示"用户名或密码错误"。
```

### 密码不一致
```
在注册页面，密码填 password123，确认密码填 different，验证是否提示"两次密码输入不一致"。
```

### 密码太短
```
在注册页面尝试使用 3 位密码，验证是否提示"密码强度不足"。
```

### 邮箱格式错误
```
在注册页面使用无效邮箱格式（如"test"），验证是否提示"邮箱格式不正确"。
```

### 用户已存在
```
尝试注册已存在的用户名"test"，验证是否提示"用户已存在"。
```
