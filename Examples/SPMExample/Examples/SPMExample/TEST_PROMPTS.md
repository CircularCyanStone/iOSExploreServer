# iOS 自动化测试提示词集合

## 测试目标

使用 MCP 和 Skills 测试新创建的登录流程模块，验证完整的用户注册、登录、重置密码、退出登录流程。

---

## 测试准备

### 1. 启动测试环境

```
请帮我启动 SPMExample 模拟器并显示登录界面。

执行步骤：
1. 使用 sim-app profile
2. 构建并运行应用
3. 启动时传入 IOS_EXPLORE_SHOW_LOGIN=1 环境变量
4. 验证 server 是否正常启动（curl ping）
```

---

## 场景 1：成功登录流程

### 1.1 使用预置账号登录

```
请帮我测试登录功能，使用预置的测试账号：
- 用户名：test
- 密码：123456

具体步骤：
1. 在登录页面的用户名输入框填写 "test"
2. 在密码输入框填写 "123456"
3. 点击登录按钮
4. 等待网络请求完成（大约 2 秒）
5. 验证是否跳转到首页
6. 截图确认当前页面
```

### 1.2 验证首页内容

```
验证登录成功后的首页，检查以下内容：
1. 页面标题是否显示"首页"
2. 用户名标签是否显示"test"
3. 邮箱标签是否显示正确的邮箱
4. 是否有"退出登录"按钮
```

## 场景 4：错误处理测试

### 4.1 空输入验证

```
测试空输入验证：

1. 在登录页面，不填写任何内容，直接点击登录按钮
2. 验证是否显示"请输入用户名"的错误提示
3. 填写用户名"test"，密码留空，点击登录
4. 验证是否显示"请输入密码"的错误提示
```

### 4.2 密码错误

```
测试密码错误场景：

1. 输入用户名：test
2. 输入错误密码：wrongpassword
3. 点击登录按钮
4. 等待请求完成
5. 验证是否显示"用户名或密码错误"的提示
```

### 4.3 密码不一致

```
测试注册时密码不一致的场景：

1. 在登录页点击"去注册"
2. 填写注册信息：
   - 用户名：testuser3
   - 邮箱：testuser3@example.com
   - 密码：password123
   - 确认密码：different123
3. 点击注册按钮
4. 验证是否显示"两次密码输入不一致"的错误提示
```

### 4.4 密码强度不足

```
测试密码强度验证：

1. 在注册页面尝试注册短密码
2. 填写信息：
   - 用户名：testuser4
   - 邮箱：testuser4@example.com
   - 密码：123
   - 确认密码：123
3. 点击注册按钮
4. 验证是否显示"密码强度不足（至少6位）"的错误提示
```

### 4.5 邮箱格式错误

```
测试邮箱格式验证：

1. 在注册页面填写无效邮箱
2. 填写信息：
   - 用户名：testuser5
   - 邮箱：invalidemail（无效格式）
   - 密码：password123
   - 确认密码：password123
3. 点击注册按钮
4. 验证是否显示"邮箱格式不正确"的错误提示
```

### 4.6 用户已存在

```
测试重复注册：

1. 尝试注册已存在的用户名"test"
2. 填写信息：
   - 用户名：test
   - 邮箱：test2@example.com
   - 密码：password123
   - 确认密码：password123
3. 点击注册按钮
4. 验证是否显示"用户已存在"的错误提示
```

## 场景 7：UI 状态测试

### 7.1 Loading 状态

```
观察并验证 Loading 状态：

1. 输入用户名和密码
2. 点击登录按钮
3. 在请求完成前（1.5秒内）观察：
   - 登录按钮是否显示 loading 指示器
   - 登录按钮文字是否消失
   - 按钮是否被禁用
4. 请求完成后验证按钮恢复正常
```

### 7.2 错误提示显示/隐藏

```
测试错误提示的显示和隐藏：

1. 输入错误的密码触发错误提示
2. 验证错误标签是否可见
3. 修改输入内容后重新点击登录
4. 验证旧的错误提示是否被清除
```

## 测试验收标准

测试完成后，验证以下内容：

### ✅ 功能验收
- [ ] 可以使用预置账号登录
- [ ] 可以注册新用户
- [ ] 可以重置密码
- [ ] 可以正常退出登录
- [ ] 所有导航流程正常

### ✅ 错误处理验收
- [ ] 空输入有验证提示
- [ ] 错误密码有提示
- [ ] 密码不一致有提示
- [ ] 邮箱格式错误有提示
- [ ] 重复用户名有提示

### ✅ UI 状态验收
- [ ] Loading 状态正常显示
- [ ] 错误提示正确显示/隐藏
- [ ] 按钮状态正确切换

### ✅ 日志验收
- [ ] 所有操作都有日志输出
- [ ] 日志格式正确（带表情符号）
- [ ] 成功/失败日志都有记录
